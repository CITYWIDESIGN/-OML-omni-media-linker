# Omni Media Linker

[English](README.md)

Omni Media Linker 是一个 Windows 桌面工具，用来把全局快捷键精确绑定到指定的媒体窗口。

它不依赖系统当前的“媒体焦点”，而是让你明确控制你想控制的那个播放器：

- `Alt+1` 可以暂停视频
- `Alt+2` 可以切歌
- 不同播放器可以绑定不同快捷键，互不干扰

## 解决了什么问题

传统媒体键通常取决于当前哪个应用拿到了媒体焦点，这会带来很多不确定性：

- 响应的是错误的播放器
- 按了没反应
- 音乐和视频之间互相抢控制权

Omni Media Linker 的做法是把媒体指令直接投递给你选定的目标窗口，从而实现更稳定的“点对点”控制。

## 核心功能

- 扫描当前活跃的桌面窗口
- 建立“目标窗口 + 动作 + 全局快捷键”的绑定关系
- 支持 `启用`、`禁用`、`编辑`、`删除` 绑定
- 将绑定关系持久化到本地 `ini` 配置文件
- 注册全局热键并在后台消息循环中分发动作
- 阻止程序被重复启动，避免多实例冲突

## 支持的媒体动作

- 播放 / 暂停
- 下一曲
- 上一曲
- 停止
- 音量加
- 音量减
- 静音

## 匹配模式

每条绑定都可以选择一种窗口匹配模式：

1. `标题精确匹配`
按窗口标题和类名精确匹配，适合标题稳定的程序。

2. `标题包含匹配`
当前窗口标题只要包含保存时的标题文本就算匹配，适合标题会部分变化的网页播放器。

3. `进程名匹配`
按可执行文件名匹配，适合像 Spotify 这种窗口标题变化很大的应用。

4. `进程名 + 类名匹配`
按可执行文件名和窗口类名共同匹配，适合需要更严格区分窗口类型的场景。

## 项目结构

- `scanner`
负责扫描活跃窗口，并采集标题、类名、进程 ID、进程名等信息。

- `actions`
负责向目标窗口发送媒体控制相关的 Windows 消息。

- `mapper`
负责维护热键与目标绑定之间的映射关系。

- `config_loader`
负责用 `ini` 格式加载和保存绑定配置。

- `dispatcher`
负责注册全局热键，并在 Windows 消息循环中分发动作。

- `main`
负责提供 CLI 界面并串联整个程序流程。

## 典型使用流程

1. 启动程序
2. 扫描可见窗口
3. 选择目标窗口
4. 选择动作
5. 选择匹配模式
6. 录入快捷键
7. 保存绑定
8. 启用绑定并启动监听
